# Forces du projet

## Architecture
- Séparation frontend / backend / base de données
- Utilisation de DAO pour structurer les accès aux données

## Fonctionnel
- Parcours utilisateur complet (catalogue → panier → paiement)
- Intégration d’un système de paiement réel (Stripe)

## Automatisation
- Utilisation de Raindrop + RSS + Make pour la veille
- Centralisation des sources dans NotebookLM

## Sécurité
- Gestion des sessions utilisateurs
- Validation des données
- Paiement sécurisé via Stripe

## Expérience utilisateur
- Interface simple et claire
- Navigation fluide entre les pages
